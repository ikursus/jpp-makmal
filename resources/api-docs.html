01 — Pengenalan

Gambaran Keseluruhan

Semua endpoint berada di bawah awalan /api/v1 dan memulangkan JSON. Kecuali login, setiap permintaan memerlukan token Bearer Sanctum. Aplikasi ini direka untuk pengguna — admin terus guna panel web.

JSON sahaja Bearer token (Sanctum) Token-per-peranti 422 envelope konsisten
Ganti __APP_URL__ dengan host pelayan anda. Sertakan header Accept: application/json pada setiap permintaan supaya ralat dipulangkan dalam bentuk JSON.
02 — Aliran

Pengesahan

Pengguna hanya menaip email + kata laluan. Pelayan memulangkan satu token unik yang aplikasi simpan dan lampirkan pada setiap permintaan seterusnya. Pengguna tidak pernah melihat atau menaip token.

  1. POST /login dengan email & kata laluan → terima token.
  2. Simpan token secara selamat dalam peranti (cth. flutter_secure_storage).
  3. Lampirkan pada setiap permintaan: header Authorization: Bearer <token>.
  4. POST /logout membatalkan token peranti semasa sahaja.

06 — Integrasi

Integrasi Flutter

Contoh menggunakan dio + flutter_secure_storage. Interceptor melampirkan token secara automatik — pengguna tidak menyentuhnya.

pubspec.yamldependencies:
  dio: ^5.4.0
  flutter_secure_storage: ^9.0.0
api_client.dart · interceptor tokenimport 'package:dio/dio.dart';
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

class ApiClient {
  ApiClient() {
    dio = Dio(BaseOptions(
      baseUrl: '__APP_URL__/api/v1',
      headers: {'Accept': 'application/json'},
    ));

    dio.interceptors.add(InterceptorsWrapper(
      onRequest: (options, handler) async {
        final token = await _storage.read(key: 'token');
        if (token != null) {
          options.headers['Authorization'] = 'Bearer $token';
        }
        handler.next(options);
      },
    ));
  }

  late final Dio dio;
  final _storage = const FlutterSecureStorage();
}
auth_service.dart · login / logoutclass AuthService {
  AuthService(this._client);
  final ApiClient _client;
  final _storage = const FlutterSecureStorage();

  Future<void> login(String email, String password) async {
    final res = await _client.dio.post('/login', data: {
      'email': email,
      'password': password,
      'device_name': 'flutter-app',
    });
    await _storage.write(key: 'token', value: res.data['token']);
  }

  Future<void> logout() async {
    await _client.dio.post('/logout');
    await _storage.delete(key: 'token');
  }
}

Future<List<dynamic>> fetchItems({String? search}) async {
  final res = await _client.dio.get('/items', queryParameters: {
    if (search != null && search.isNotEmpty) 'search': search,
  });
  return res.data['data'];
}

// Hantar permohonan pinjaman
Future<Map<String, dynamic>> submitApplication({
  required List<Map<String, int>> items, // [{ 'item_id': 1, 'quantity': 2 }]
  required String startDate,
  required String endDate,
  required String purpose,
}) async {
  final res = await _client.dio.post('/loan-applications', data: {
    'items': items,
    'start_date': startDate,
    'end_date': endDate,
    'purpose': purpose,
  });
  return res.data['data']; // 201 Created
}

// Senarai permohonan sendiri
Future<List<dynamic>> fetchMyApplications() async {
  final res = await _client.dio.get('/loan-applications');
  return res.data['data'];
}
07 — Rujukan

Pengendalian Ralat

Setiap respons 422 menggunakan envelope yang sama, jadi klien boleh mengendalikan ralat secara seragam.

422 · envelope{
  "message": "...",
  "errors": { "<medan>": [ "..." ] }
}
KodMaksud
200 / 201Berjaya
422Validasi gagal · stok tak cukup (errors.items) · tiada daerah (errors.district)
401Token tiada atau tidak sah
403Mengakses permohonan pengguna lain
404Item / permohonan tidak ditemui (atau item tidak dalam katalog)
429Had cubaan log masuk dilampaui (5 / 15 minit)
